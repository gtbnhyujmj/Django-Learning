
1. 資料庫 (models.py) 是用來「儲存和管理資料」
你用 models.py 定義資料表結構，存放幣種、訊號、價格等資料

資料庫裡的資料是動態的，隨時會變動，並且是後端存取的核心

例如：你有一筆 Signal 紀錄，會存在資料庫裡

2. {% block %} ... {% endblock %} 是 Django 模板語法，負責「網頁前端畫面排版和內容呈現」
用在 HTML 模板中，控制區塊內容如何被繼承和替換

例如：base.html 定義一個 block 首頁，其他子模板填入實際內容

這是前端的畫面組合邏輯，跟資料庫無直接關係

3. 兩者要配合，但不會互相替代
資料庫存資料 → Django view 撈資料 → 傳給模板 → {% block %} 呈現畫面

你不會用 {% block %} 取代資料庫，兩者是不同層的東西，各司其職

簡單比喻
角色	角色說明
資料庫	書櫃，放資料的地方
模板區塊	書本的章節標題與段落結構
Django view	圖書館管理員，拿書給讀者
